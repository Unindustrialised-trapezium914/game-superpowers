# game-spec-driven-planning

Part of the **Game Superpowers** skill system.

## Summary

Use when the user wants a persistent requirements/spec/design/tasks workflow, asks for OpenSpec-style planning, or needs durable change artifacts in the target repo.

## Recommended usage

- Use `game-spec-driven-planning` when the user explicitly wants `requirements/spec/design/tasks` artifacts in the repo.
- Apply `game-spec-driven-planning` when a game feature needs durable change records that survive across sessions.
- Use `game-spec-driven-planning` on top of the normal Build, Repair, Audit, or Polish route instead of replacing those tracks.
