# Project State and Risk

## Greenfield
Prefer larger coherent AI-native steps. Do not prematurely optimize for human ticket granularity.

## Existing but prelaunch
Broad refactors are acceptable if they materially improve architecture, feel, or delivery confidence.

## Live / shipped
Default to constrained changes unless the user explicitly authorizes a larger migration.

## Core decision
Always classify:
- is this from scratch?
- is it already in production or exposed to real users?
- are large refactors allowed?
